# Contributing to GenText

I welcome and appreciate any contributions that'll make GenText even better. 
Want to report a bug? Or suggest an enhancement? Or even contribute code? 
Read further as this document will guide you through the process.


## How to Contribute

1. **Create an Issue**: If you encounter a bug, have a feature request, or would like to propose a change, please [open an issue](https://github.com/lifewjola/gentext/issues) on the GitHub repository. Be sure to check if a similar issue or pull request already exists.

2. **Fork the Repository**: If you plan to contribute code, start by forking the GenText repository to your GitHub account.

3. **Clone the Repository**: Next, clone your forked repository to your local development environment:

   ```bash
   git clone https://github.com/your-username/gentext.git
   cd gentext
   ```

4. **Create a Branch**: Create a new branch for your work. Name the branch descriptively to indicate its purpose. For example:

   ```bash
   git checkout -b feature/my-new-feature
   ```

5. **Make Changes**: Make your desired changes or additions to the code. Ensure that your code follows the project's coding conventions and style.

6. **Test Your Changes**: Before submitting your changes, run the appropriate tests to ensure your code doesn't introduce new issues.

7. **Commit Your Changes**: Commit your changes with a clear and concise commit message that explains what your changes accomplish. For example:

   ```bash
   git commit -m "Add feature: XYZ"
   ```

8. **Push Your Changes**: Push your changes to your GitHub fork:

   ```bash
   git push origin feature/my-new-feature
   ```

9. **Create a Pull Request**: Once your changes are committed and pushed to your fork, [create a pull request](https://github.com/lifewjola/gentext/compare) on the GenText repository. Provide a clear description of your changes, why they are necessary, and any related issues.

10. **Review and Collaboration**: I will review your pull request (You may need to make additional changes or answer questions during the review process).

11. **Approval and Merge**: Once your pull request is approved and passes all checks, it will be merged into the main branch.

12. **Celebrate**: Congratulations! You've successfully contributed to GenText. Thank you for your valuable contribution.
